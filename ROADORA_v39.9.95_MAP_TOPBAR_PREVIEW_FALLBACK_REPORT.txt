Roadora v39.9.95 Map Topbar Preview Fallback v1

Aanleiding
- Trajecten toont km/tijd weer correct.
- Kaart-topbar bleef leeg, vooral bij testen via VS Code/Live Server.
- v39.9.94 koppelde de topbar aan de stabiele ORS-resolver, maar miste de preview/fallback-bron die Trajecten al gebruikt.

Oorzaak
- Trajecten gebruikt naast ORS ook een preview/fallback-total.
- De kaart-topbar gebruikte alleen ORS/stable summary.
- Bij lokale VS Code tests zonder /api/route is er vaak geen ORS-summary, maar wel een fallback/preview-route.

Fix
- resolveMapTopbarSummaryV39994 behoudt eerst ORS/stable summary.
- Als die ontbreekt, gebruikt de topbar de huidige routeCoordinates-afstand als preview.
- Als ook die ontbreekt, valt hij terug op dezelfde simpele preview-benadering als Trajecten.

Niet aangepast
- Geen CSS gewijzigd.
- Geen ORS/backend gewijzigd.
- Geen kaart-drawing gewijzigd.
- Geen Stops/Nu Nodig/Meer/sheets/popovers gewijzigd.

Testadvies
1. Routeplan -> Plan route.
2. Test lokaal in VS Code/Live Server.
3. Controleer kaart-topbar: ETA/tijd + afstand mogen niet leeg blijven.
4. Controleer Trajecten: km/tijd blijven staan.
5. Controleer op Vercel: ORS-summary blijft leidend boven preview.
